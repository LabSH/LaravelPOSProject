# 🛒 Laravel 기반 주문 관리 시스템 개발
주문 관리를 효율적으로 구현하는 것은 POS 시스템 개발의 핵심 요소 중 하나라고 생각했습니다. 이에 따라 Laravel 프레임워크를 활용하여 주문·회원 관리 시스템을 직접 설계하고 구현했습니다.
`Artisan CLI`와 `Composer`를 이용해 프로젝트 환경을 빠르게 구성하고, 마이그레이션과 시딩을 통해 초기 데이터를 준비했습니다.
이 프로젝트에서는 CRUD 기능을 넘어, 관리자와 일반 사용자 간의 권한 분리, 주문 처리 로직, 데이터 무결성 유지 등 실제 서비스 개발에 필요한 백엔드 핵심 기능을 중점적으로 구현하고자 했습니다.

# 🎯 프로젝트 목표
- **PHP 프레임워크 기초 학습 및 실습**: Laravel 프레임워크의 기본 구조와 동작 원리를 이해하고, `라우팅` `컨트롤러` `Eloquent ORM` `미들웨어` 등 핵심 기능을 실습하며 백엔드 개발 역량을 쌓습니다.
- **MVC 아키텍처 익히기**: `모델(Model)` `뷰(View)` `컨트롤러(Controller)`의 역할과 책임을 명확히 구분하여 개발하며, 팀 협업과 유지보수에 유리한 코드 구조를 학습합니다.
- **Blade 템플릿 활용**: Laravel의 Blade 템플릿 엔진을 사용해 동적 UI를 구현하며, 데이터 바인딩과 조건문, 반복문을 활용해 사용자 친화적인 화면을 구성하는 방법을 학습합니다.
- **기본적인 권한 관리 구현**: 사용자 유형(관리자/일반 사용자)에 따라 주문 데이터 접근 권한을 구분하여, 역할 기반의 간단한 권한 제어 시스템을 구현하며 보안과 사용자 경험의 중요성을 이해합니다.
- **사용자 기능:** 사용자는 로그인 후 상품을 조회하고, 원하는 상품을 주문할 수 있는 기능을 사용할 수 있습니다.
- **관리자 기능:** 특정 권한(gubun 값이 1)을 가진 사용자는 로그인 시 관리자 페이지에 접근할 수 있으며, 상품 등록·수정·삭제, 주문 관리, 사용자 정보 관리 등의 기능을 수행할 수 있습니다. 특히 주문 조회 화면은 관리자 사용성을 고려하여 사용자 친화적으로 설계합니다.

# 🛠️ 주요 기능 및 특징
**관리자 프로그램**
- 주문 처리: 실시간으로 사용자 주문을 확인하고 처리 상태를 업데이트하여 효율적인 주문 관리를 지원합니다.
- 제품 관리: 상품 등록, 수정, 삭제 기능을 통해 제품 데이터베이스를 직관적으로 관리합니다.
- 회원 관리: 사용자 정보를 저장하고, 관리자 및 일반 사용자 간 권한을 설정하여 접근 제어를 구현합니다.

**사용자 프로그램**
- 로그인/회원가입: 세션을 통해 사용자가 시스템에 접근하고 사용자 서비스를 이용할 수 있도록 합니다.
- 간편한 상품 주문: 사용자 친화적인 인터페이스로 상품을 쉽게 탐색하고 주문을 완료할 수 있도록 지원합니다.

**Blade 템플릿 활용**
- 데이터 바인딩 및 보안: Blade의 `{{ }}` 문법을 사용해 백엔드 데이터를 안전하게 이스케이프하여 XSS 공격을 방지합니다.
- 폼 입력 편의성: `old()` 헬퍼를 활용해 폼 재제출 시 사용자 입력을 복원하여 사용성을 향상시킵니다.


# 🔍 문제 해결 및 사례

**라우트 설정:** 초기 라우트 설정 시 `Route::resource`의 역할과 CRUD 라우트 구성을 이해하지 못해 적절한 라우팅 설계가 어려웠습니다 하지만 공식 문서와 자료를 참고해 `Route::resource`가 생성하는 CRUD 라우트를 학습하고, 관리자 및 사용자별 라우트를 체계적으로 구성했습니다.

**DB 설계**: 초기 DB 설계 시 필요한 컬럼과 설정을 명확히 정의하지 않아, 주문 관리 시스템의 테이블 수정이 빈번히 발생했습니다. 이런 수정을 방지하기 위해 ERD를 작성하며 테이블간 구조를 체계적으로 설계하고, 마이그레이션을 활용해 테이블 구조를 재구성하며 개발했습니다.

**데이터 유효성**: 주문 생성 시 유효하지 않은 입력값(예: 수량 0)이 데이터베이스에 저장되었습니다. 하지만 `Validator`를 사용해 컨트롤러에서 입력값을 검증하고, 실패 시 사용자에게 에러 메시지를 반환하며 데이터의 유효성을 검증하였습니다.


# 📚 성과 및 배운 점

**Laravel 프레임워크 실습과 PHP 응용 능력 향상**: Laravel의 핵심 기능(라우팅, 컨트롤러, Eloquent ORM, 미들웨어, Blade 템플릿)을 직접 실습하며, PHP를 활용한 주문 관리 시스템을 구현했습니다. `Artisan CLI`와 `Composer`를 통해 개발 환경을 구성하고, 공식 문서, 전공 서적을 참고하여 프레임워크의 구조와 동작 원리를 체계적으로 익혔습니다.

**MVC 패턴의 실제 적용 경험**: Laravel의 MVC 아키텍처를 처음 접하며, `모델(Model)` `뷰(View)` `컨트롤러(Controller)`의 역할을 명확히 분리하여 개발하는 방식을 학습했습니다. Blade 템플릿으로 뷰를 구성하고, 컨트롤러에 비즈니스 로직을 집중시켜 유지보수성과 가독성을 높이는 코드 구조를 경험했습니다.

**입력값 유효성 검증과 사용자 경험 개선**: 주문 생성 시 유효하지 않은 입력값(예: 수량 0)을 방지하기 위해 `Validator`를 활용한 서버단 유효성 검증을 구현했습니다. 검증 실패 시에는 Blade를 통해 사용자에게 에러 메시지를 제공하며, 사용자 경험과 데이터 무결성의 균형을 맞추는 법을 배웠습니다.

**초기 DB설계의 중요성 경험**: 초반에 데이터 구조를 명확히 정의하지 않아 수정이 잦았던 경험을 통해, 개발 전 자료 흐름과 구조를 충분히 고민하고 정리하는 설계의 중요성을 실감했습니다. 이 경험은 이후의 개발 방향과 안정성 확보에 큰 도움이 되었습니다.

**라우팅과 엔드포인트 설계에 대한 이해**: `Route::resource`의 자동 CRUD 라우트 생성 기능을 제대로 이해하지 못했던 초기 시행착오를 겪으며, 공식 문서와 예제 코드를 참고해 관리자와 사용자 역할별로 라우트를 체계적으로 구성했습니다. 또한, 미들웨어를 활용한 권한 기반 접근 제어를 구현하며 보안 측면에서의 라우팅 설계 중요성도 인식하게 되었습니다.
